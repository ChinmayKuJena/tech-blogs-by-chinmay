Exponential Backoff with Jitter for Reliable API Calls

📅 2026-05-07
Learn why plain retries can hammer downstream services and how exponential backoff with jitter spreads load, reduces thundering herd, and improves reliability.
typescript retry backend performance

Insight

When a service call fails transiently (network glitch, rate‑limit, temporary outage), a naïve retry loop often retries immediately or with a fixed delay. This can create a thundering herd where many instances hammer the same endpoint, worsening latency and potentially causing cascading failures. Exponential backoff increases the wait time after each attempt, while adding random jitter prevents synchronized retries across instances. The pattern is lightweight, language‑agnostic, and dramatically improves resilience without extra infrastructure.

Example

async function fetchWithBackoff(url: string, maxAttempts = 5) {
  let attempt = 0;
  while (true) {
    try {
      return await fetch(url);
    } catch (err) {
      if (++attempt >= maxAttempts) throw err;
      const base = 100; // 100ms
      const jitter = Math.random() * base;
      const delay = Math.pow(2, attempt) * base + jitter;
      await new Promise(r => setTimeout(r, delay));
    }
  }
}

Takeaway

Wrap flaky calls in an exponential‑backoff‑with‑jitter loop; it’s a simple, zero‑cost guard that keeps your services healthy under load and during transient failures.
