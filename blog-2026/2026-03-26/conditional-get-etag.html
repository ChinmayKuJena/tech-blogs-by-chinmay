Conditional GET with ETag: Reduce Unnecessary Payloads

📅 2026-03-26
Use ETag headers and conditional GET to let clients skip sending data when nothing changed, saving bandwidth and latency.
http caching backend undefined

Insight

Many developers assume every GET request must return the full resource, even when the client already has the latest version. This wastes bandwidth and adds latency. By leveraging HTTP's ETag header and the If-None-Match request header, servers can tell the client "nothing changed" with a lightweight 304 response. The myth that ETags are hard to manage disappears once you see a few lines of Express code.

Example

import express from 'express';
import crypto from 'crypto';
import { getUserById } from './service';

const app = express();
app.get('/users/:id', async (req, res) => {
  const user = await getUserById(req.params.id);
  const etag = crypto.createHash('sha1').update(JSON.stringify(user)).digest('hex');
  if (req.headers['if-none-match'] === etag) {
    return res.status(304).end();
  }
  res.set('ETag', etag).json(user);
});

Takeaway

Add an ETag to any cache‑able response; if the client sends If-None-Match and it matches, return 304. This simple pattern can cut down on data transfer for high‑traffic read endpoints without extra infrastructure.
