OpenTelemetry Tracing in FastAPI: Quick Setup for End‑to‑End Visibility

📅 2026-03-19
Add distributed tracing to a FastAPI service in minutes using OpenTelemetry, giving you request‑level visibility without heavy instrumentation.
fastapi observability opentelemetry python

Insight

FastAPI’s async nature makes it an excellent candidate for distributed tracing, yet many teams skip it because they think the setup is heavyweight. OpenTelemetry provides a tiny, auto‑instrumented package that captures inbound HTTP requests, outbound HTTP calls, and database queries with a single line of code. The real benefit appears when you combine it with a backend like Jaeger or Grafana Tempo: you instantly get request latency breakdowns, error hotspots, and service‑to‑service call graphs, all without rewriting business logic.

Example

# main.py
from fastapi import FastAPI
from opentelemetry.instrumentation.fastapi import FastAPIInstrumentor
from opentelemetry.instrumentation.requests import RequestsInstrumentor
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.jaeger.thrift import JaegerExporter

app = FastAPI()

# Set up tracer provider and exporter
provider = TracerProvider()
jaeger = JaegerExporter(agent_host_name="jaeger", agent_port=6831)
provider.add_span_processor(BatchSpanProcessor(jaeger))

# Register global tracer provider
from opentelemetry import trace
trace.set_tracer_provider(provider)

# Auto‑instrument FastAPI and outgoing HTTP calls
FastAPIInstrumentor().instrument_app(app)
RequestsInstrumentor().instrument()

@app.get("/ping")
async def ping():
    return {"msg": "pong"}

Takeaway

Add the two‑line instrumentation (FastAPIInstrumentor().instrument_app(app)) and a Jaeger exporter, and you’ll start seeing end‑to‑end traces in seconds. Remember to run a collector (Jaeger/Grafana Tempo) alongside your service; the low‑overhead OpenTelemetry SDK ensures production performance stays unaffected.
