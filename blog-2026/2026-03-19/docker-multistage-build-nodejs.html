Slimming Down Node.js Containers with Multi‑Stage Docker Builds

📅 2026-03-19
Learn how multi‑stage Docker builds can dramatically reduce the size of Node.js images, improve startup time, and eliminate unnecessary build tools from production containers.
docker devops nodejs undefined

Insight

Multi‑stage builds let you separate the heavy build environment from the lightweight runtime image. By compiling TypeScript, installing only production dependencies, and discarding the build tools, you can shrink a typical Node.js image from >200 MB to <50 MB, which speeds up deployments and reduces attack surface.

Example

# ---- Build Stage ----
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json tsconfig.json ./
RUN npm ci
COPY src ./src
RUN npm run build   # compiles TypeScript to ./dist

# ---- Runtime Stage ----
FROM node:20-alpine AS runtime
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY package*.json ./
RUN npm ci --only=production
CMD ["node", "dist/index.js"]

Takeaway

Always use a multi‑stage Dockerfile for Node.js services: compile in a dedicated stage, then copy only the compiled output and production deps to the final image. This yields smaller, faster, and more secure containers with minimal effort.
