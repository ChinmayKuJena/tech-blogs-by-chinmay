Efficient Pagination with SQL Window Functions

📅 2026-04-24
Learn how window functions can replace OFFSET/LIMIT for stable, performant pagination and ranking in PostgreSQL.
sql postgresql performance undefined

Insight

Traditional OFFSET/LIMIT pagination can become slow on large tables and produces inconsistent results when rows are inserted or deleted between requests. Window functions like ROW_NUMBER() let you compute a stable row index on the fly, enabling keyset‑style pagination without the cost of scanning skipped rows. This approach also makes it trivial to add ranking columns for leaderboards or ordered feeds.

Example

-- Fetch page 3 (rows 41‑60) for a specific customer, ordered by newest first
SELECT *
FROM (
  SELECT *, ROW_NUMBER() OVER (ORDER BY created_at DESC) AS rn
  FROM orders
  WHERE customer_id = $1
) sub
WHERE rn BETWEEN $2 AND $3;  -- $2 = 41, $3 = 60

The inner query assigns a sequential number to each row based on created_at. The outer query then filters by that number, avoiding the heavy OFFSET scan.

Takeaway

When building paginated APIs, prefer window‑function‑based keyset pagination over OFFSET/LIMIT. It scales better, stays consistent under concurrent writes, and gives you ranking for free.
