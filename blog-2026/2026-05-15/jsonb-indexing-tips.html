Speed Up JSONB Queries with Targeted Indexes

📅 2026-05-15
Learn how partial GIN and BTREE indexes on JSONB columns can turn slow document scans into lightning‑fast lookups.
postgresql database performance undefined

Insight

PostgreSQL’s JSONB type is great for schemaless data, but without the right index queries can degenerate into full table scans. A generic GIN index (jsonb_path_ops) covers all keys, yet it’s bulky and slower for selective lookups. By creating partial GIN indexes on the exact paths you query, or even BTREE indexes on extracted scalar values, you let the planner prune rows dramatically.

Example

-- GIN index for any key under “metadata”
CREATE INDEX idx_events_meta ON events USING GIN (metadata jsonb_path_ops);

-- Partial GIN index for only events where type = 'click'
CREATE INDEX idx_events_click_type ON events USING GIN (metadata jsonb_path_ops)
WHERE metadata->>'type' = 'click';

-- BTREE index on a scalar extracted from JSONB
CREATE INDEX idx_events_user_id ON events ((metadata->>'user_id'));

Takeaway

Identify the hot JSONB paths, then apply a focused partial GIN or BTREE index. This cuts query time from seconds to milliseconds while keeping index size modest.
