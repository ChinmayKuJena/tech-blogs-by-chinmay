Window Functions: Powerful Tools for Ranking and Pagination

📅 2026-05-11
Leverage SQL window functions to implement efficient ranking, running totals, and cursor‑free pagination without costly subqueries.
sql postgresql performance undefined

Insight

Window functions let you compute row‑level aggregates without collapsing rows, making them ideal for ranking, running totals, and pagination. Unlike traditional OFFSET/LIMIT, they avoid skipping rows on each request, which scales better on large tables. By using ROW_NUMBER() or RANK() you can generate stable page cursors based on business keys, and SUM() OVER provides cumulative totals in a single pass.

Example

-- Get a page of users ordered by score, with a stable cursor
WITH ranked AS (
  SELECT id, username, score,
         ROW_NUMBER() OVER (ORDER BY score DESC) AS rn
  FROM users
)
SELECT id, username, score
FROM ranked
WHERE rn > $1 AND rn <= $1 + $2; -- $1 = offset, $2 = page size

This query returns a deterministic slice without re‑scanning the entire table for each page.

Takeaway

Prefer window functions for pagination and ranking over OFFSET/LIMIT when dealing with large, frequently‑updated datasets. They keep the query planner happy and give you stable, performant cursors.
