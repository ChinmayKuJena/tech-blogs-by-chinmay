Atomic Rate Limiting with Redis Lua Scripts

📅 2026-03-29
Learn how to implement a precise, distributed rate limiter using a single Redis Lua script, guaranteeing atomicity and low latency across multiple API instances.
redis rate-limiting typescript backend

Insight

Rate limiting protects your APIs from abuse, but naïve implementations that read‑modify‑write a counter can race when multiple instances serve traffic. Redis excels at fast key‑value storage, and its Lua scripting engine runs commands atomically on the server, eliminating race conditions without extra coordination layers. By encapsulating the token‑bucket logic in a short script, you get sub‑millisecond checks that work consistently across a horizontally‑scaled fleet.

Example

import Redis from 'ioredis';
const redis = new Redis();

// KEYS[1] = bucket key, ARGV[1] = limit, ARGV[2] = window (seconds)
const lua = `
local current = redis.call('INCR', KEYS[1])
if tonumber(current) == 1 then
  redis.call('EXPIRE', KEYS[1], ARGV[2])
end
if tonumber(current) > tonumber(ARGV[1]) then
  return 0
else
  return 1
end`
;
export async function allowRequest(id: string, limit: number, windowSec: number): Promise<boolean> {
  const result = await redis.eval(lua, 1, `rl:${id}`, limit, windowSec);
  return result === 1;
}

Takeaway

Store the script SHA in your code and call EVALSHA after the first load to avoid re‑sending the script on every request—this reduces network overhead and keeps your rate limiter ultra‑lightweight.
