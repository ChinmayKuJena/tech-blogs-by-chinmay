Rate Limiting with Redis Lua Scripts

📅 2026-04-29
Learn how a single Redis Lua script can enforce precise request throttling without race conditions, keeping your API safe from abuse.
redis rate-limiting lua backend

Insight

Implementing rate limiting at the API gateway often requires a distributed lock to avoid race conditions across multiple instances. Redis' atomic Lua scripting lets you evaluate and update a counter in a single round‑trip, guaranteeing consistency even under high concurrency. By storing a sliding‑window counter per client key, you can enforce strict request caps without external coordination.

Example

import { createClient } from 'redis';
const client = createClient();
await client.connect();

// Lua script: increment counter, set TTL on first hit, return remaining quota
const script = `
local key = KEYS[1]
local limit = tonumber(ARGV[1])
local ttl = tonumber(ARGV[2])
local current = tonumber(redis.call('incr', key))
if current == 1 then redis.call('expire', key, ttl) end
if current > limit then return -1 else return limit - current end
`;

export async function checkRateLimit(clientId: string, limit = 100, windowSec = 60) {
  const remaining = await client.eval(script, { keys: [clientId], arguments: [limit, windowSec] });
  if (remaining < 0) throw new Error('Rate limit exceeded');
  return remaining;
}

Takeaway

Use a single Redis Lua script for rate limiting to keep the operation atomic and latency low. This pattern scales horizontally, eliminates race conditions, and requires only one Redis round‑trip per request.
