ETag‑Based Conditional GETs: Reduce Bandwidth and Improve Cache Freshness

📅 2026-05-05
Leverage HTTP ETag headers in Express to let clients cache responses safely and avoid unnecessary data transfer.
http express caching typescript

Insight

When a client repeatedly requests the same resource, sending the full payload each time wastes bandwidth and CPU. HTTP's ETag header lets the server provide a fingerprint of the current representation. On subsequent calls the client includes If-None-Match; the server can reply with 304 Not Modified if the content hasn't changed, keeping the response lightweight and preserving cache freshness.

Example

import express from 'express';
import crypto from 'crypto';
import { readFileSync } from 'fs';

const app = express();
const data = readFileSync('static/data.json', 'utf8');
const etag = crypto.createHash('md5').update(data).digest('hex');

app.get('/data', (req, res) => {
  if (req.headers['if-none-match'] === etag) {
    return res.status(304).end();
  }
  res.set('ETag', etag).json(JSON.parse(data));
});

app.listen(3000);

Takeaway

Generate a stable ETag (hash or version) for immutable or rarely‑changing responses and honor If-None-Match. This simple pattern cuts down on network traffic, speeds up client rendering, and scales effortlessly without extra infrastructure.
