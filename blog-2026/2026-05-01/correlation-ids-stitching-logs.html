Correlation IDs: Stitching Together Logs Across Services

📅 2026-05-01
Learn why a simple correlation ID header is essential for tracing requests across microservices and how to implement it with a tiny NestJS middleware.
backend typescript observability logging

Insight

When a request hops through multiple microservices, each component writes its own logs. The common myth is that timestamps alone let you piece together a request's journey. In practice, without a shared identifier, you end up guessing which log lines belong together, making root‑cause analysis painful and slow. Propagating a lightweight correlation ID via an HTTP header gives you a deterministic key to group logs, measure end‑to‑end latency, and debug failures across service boundaries.

Example

// correlation-id.middleware.ts (NestJS)
import { Injectable, NestMiddleware } from '@nestjs/common';
import { v4 as uuidv4 } from 'uuid';

@Injectable()
export class CorrelationIdMiddleware implements NestMiddleware {
  use(req: any, res: any, next: () => void) {
    const id = req.headers['x-correlation-id'] ?? uuidv4();
    req.correlationId = id; // attach to request object
    res.setHeader('x-correlation-id', id);
    // make it available to any logger
    console.log(`[${id}] → ${req.method} ${req.originalUrl}`);
    next();
  }
}

Register the middleware globally so every incoming request gets an ID.

Takeaway

Generate a correlation ID at the edge, forward it via X-Correlation-Id, and include it in every log statement. This tiny habit turns scattered logs into a coherent trace without any heavyweight tracing infrastructure.
